# Target Persona: Alex - Experienced Vibe Coder

## Demographics

- **Name:** Alex (archetype)
- **Role:** Non-tech founder / Tech-savvy entrepreneur
- **Experience:** 6-12+ months coding with LLM
- **Community:** "Vibe-coders" Telegram (1000+ members, Russian-speaking)

## Background

Alex heard you can build products fast with AI. Made a landing page in 1 hour with Lovable. But when building real functionality (server-side, frontend, integrations) - hit the wall.

## Pain Points

1. **Chaos:** Same request → two different contradictory solutions
2. **Time sink:** 90% debugging/refactoring vs 6% new features
3. **Context loss:** Agent "forgets" project structure after a while
4. **Cyclic dependencies:** Code becomes unmaintainable
5. **No system:** Every time reinventing the wheel

## What Alex Tried (and why it failed)

| Solution | Why it doesn't work |
|----------|---------------------|
| Cursor | Just an IDE, no methodology |
| Superpowers | Good, but less autonomous |
| Clean Architecture | Designed for humans, not LLM |
| Custom prompts | Manual labor, need to remember everything |
| Templates/Boilerplates | Static, don't adapt |

## Moment of Discovery

1. Finds DLD on HN/Reddit
2. Sees "Compare DLD with your setup" prompt
3. Runs comparison → sees the difference
4. Invests 1-2 hours (pain is big enough)
5. Runs /bootstrap → /spark → /autopilot
6. Watches 30 tool calls execute automatically
7. **WOW moment!**

## Two "WOW" Moments

1. **Intellectual:** "This diagram is more thought-through than anything I've seen"
2. **Practical:** "Autopilot shipped features without breaking deploy"

## Key Insight

Alex is NOT a beginner. Alex already "ate enough" AI coding chaos. Year+ of hype, tried everything. Ready to invest time if solution is convincing.
