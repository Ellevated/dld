# DLD Launch Strategy

## Timeline

- **Day 1:** Public launch (1 week from bootstrap)
- **Day 30:** Product Hunt (when traction + testimonials exist)

## Why NOT Product Hunt on Day 1

1. **Wrong audience:** PH = consumer products, not dev tools
2. **Heavy prep:** Need buildup (followers, reviews, comments)
3. **Competition:** Saturated platform, big marketing budgets
4. **Timing:** 12:01 AM PST launch, first 4 hours critical

> "For developer tools, Hacker News will always be more valuable - more active installs and more people asking for paid plans."

## Day 1 Channels (Priority Order)

### 1. Hacker News (Show HN) - PRIORITY #1

**Title:** `Show HN: DLD – LLM-First Architecture (90% debug → autopilot productivity)`

**Body:**
- Short pitch
- Comparison prompt
- GitHub link
- Let the tech crowd discuss organically

### 2. Vibe-coders Telegram

- 1000+ pre-warmed audience
- Russian-speaking
- Founder connection (can DM)
- Announce same day as HN

### 3. Reddit

| Subreddit | Focus |
|-----------|-------|
| r/ClaudeAI | Skills/agents angle |
| r/programming | Architecture angle |
| r/opensource | Methodology angle |

### 4. Twitter/X Thread (8-10 tweets)

1. Hook: "90% vs 6%" statistic
2. Problem description
3. Solution intro
4. Demo GIF (autopilot in action)
5. Comparison table as image
6. Real project examples
7. CTA: GitHub + comparison prompt
8. (Optional) Meme about the acronym + tag Elon

### 5. LinkedIn

- More formal tone
- Success story angle
- Personal journey narrative

### 6. Dev.to / Medium (Longform)

**Title:** "Why Clean Architecture Fails for LLM Agents"

Deep dive into principles, real examples, methodology.

## Day 30: Product Hunt

**Prerequisites:**
- Traction (stars, issues, discussions)
- Testimonials from early adopters
- Refined pitch
- Community feedback incorporated

## Killer Feature for Onboarding

### Comparison Prompt

Place prominently in README:

```
Analyze DLD methodology (github.com/desperado/dld) and compare
it with my current skills setup. Which approach will write
better code and help scale my LLM-powered project?
```

This lets Alex evaluate DLD against their existing setup BEFORE committing time.
