## Failures

### F1: Artifact sort picks old-format file over canonical (locale-dependent)

**Severity:** Minor
**Reproducibility:** Always (on en_US.UTF-8 locale)
**Expected:** `find ... | sort | tail -1` picks newest canonical file `20260318-185452-TECH-154.md`
**Actual:** Picks `2026-03-18-tech-154.md` (old-format file) because locale-aware `sort` orders `-` differently than `0`
**Steps to reproduce:**
1. Have `ai/qa/` with both old-format (`2026-03-18-tech-154.md`) and new-format (`20260318-185452-TECH-154.md`) files
2. Run: `find ai/qa -maxdepth 1 -type f -name "[0-9]*-*.md" | sort | tail -1`
3. Observe: old-format file is picked

**Evidence:**
```
=== Default locale (en_US.UTF-8) sort ===
sort | tail -1 => 2026-03-18-tech-154.md  (WRONG — old format)

=== LC_ALL=C sort ===
sort | tail -1 => 20260318-185452-TECH-154.md  (CORRECT — canonical format)
```

**User impact:** Callback may read outdated QA report for the same spec, missing the actual latest QA results. Low practical impact since old-format files will stop being generated, but problematic during transition period.
**Hint for developers:** Add `LC_ALL=C` before `sort` in pueue-callback.sh artifact scan, or switch to `ls -t | head -1` (sort by mtime).
