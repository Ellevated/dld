# BUG-083-03: Tier Numbering Alignment

**Parent:** BUG-083 | **Priority:** HIGH | **Effort:** ~2h
**Status:** queued

## Summary

Tier naming is inconsistent across create-dld CLI, setup-mcp.sh, template/CLAUDE.md, docs, and README. Users see "Tier 0/1/2" in one place and "Tier 1/2/3" in another.

## Findings Addressed

| ID | Severity | Title |
|----|----------|-------|
| CR-003 | HIGH | Tier numbering mismatch (create-dld 0/1/2 vs setup-mcp.sh 1/2/3) |
| CR-007 | LOW | CLAUDE.md tier 3 vs create-dld tier 2 mismatch |
| CR-008 | LOW | installation-guide.md tier references inconsistent |
| UX-024 | MEDIUM | Outdated model reference (Opus 4.5 vs 4.6) |

## Root Cause

Tier system evolved organically. CLI used 0-indexed (0/1/2), docs used names inconsistently, setup-mcp.sh used 1-indexed (1/2/3).

## Canonical Naming

| CLI Flag | Display Name | What's Included |
|----------|-------------|-----------------|
| `--quick` | Quick | Core CLAUDE.md, spark, no MCP |
| `--standard` | Standard | + MCP (Context7, Exa), hooks |
| `--power` | Power | + Council, autopilot, planner |

**Rule:** Never use numeric tier references (Tier 0/1/2/3). Always use names: Quick, Standard, Power.
