# BUG-083-02: Security Defaults

**Parent:** BUG-083 | **Priority:** CRITICAL | **Effort:** ~2h
**Status:** queued

## Summary

Template ships with overly permissive MCP and path handling defaults. Users inherit security risks without being aware.

## Findings Addressed

| ID | Severity | Title |
|----|----------|-------|
| SEC-002 | CRITICAL | Wildcard mcp__* grants access to ALL MCP servers |
| SEC-003 | HIGH | Path traversal in hook loading via hookName |
| SEC-009 | MEDIUM | Path traversal via CLAUDE_PROJECT_DIR |

## Root Cause

Security defaults were set for developer convenience during DLD development, then shipped to template without hardening.

## Files to Modify

### 1. `template/.claude/settings.json`
Replace `mcp__*` wildcard with explicit MCP server whitelist.

**Current:**
```json
"mcp__*": { "disabled": false }
```

**Fix:**
```json
"mcp__plugin_context7_context7__resolve-library-id": { "disabled": false },
"mcp__plugin_context7_context7__query-docs": { "disabled": false },
"mcp__exa__web_search_exa": { "disabled": false },
"mcp__exa__web_search_advanced_exa": { "disabled": false },
"mcp__exa__get_code_context_exa": { "disabled": false },
"mcp__exa__deep_search_exa": { "disabled": false },
"mcp__exa__crawling_exa": { "disabled": false },
"mcp__exa__company_research_exa": { "disabled": false },
"mcp__exa__deep_researcher_start": { "disabled": false },
"mcp__exa__deep_researcher_check": { "disabled": false }
```

### 2. `.claude/hooks/utils.mjs`
Add `getProjectDir()` validation helper to prevent path traversal via CLAUDE_PROJECT_DIR.

**Add function:**
```javascript
export function getProjectDir() {
  const dir = process.env.CLAUDE_PROJECT_DIR || process.cwd();
  const resolved = path.resolve(dir);
  // Prevent traversal outside home directory
  const home = os.homedir();
  if (!resolved.startsWith(home) && !resolved.startsWith('/tmp')) {
    return process.cwd(); // Fallback to safe default
  }
  return resolved;
}
```

### 3. `.claude/hooks/run-hook.mjs`
Validate hookName to prevent path traversal (e.g., `../../etc/passwd`).

**Add validation:**
```javascript
// Validate hookName - alphanumeric, hyphens, dots only
if (!/^[a-zA-Z0-9._-]+$/.test(hookName)) {
  process.exit(0); // Silent fail-safe for invalid hook names
}
```

### 4. Apply same changes to template copies
- `template/.claude/hooks/utils.mjs` — add getProjectDir()
- `template/.claude/hooks/run-hook.mjs` — add hookName validation

## Definition of Done

- [ ] `template/.claude/settings.json` uses explicit MCP whitelist (no wildcards)
- [ ] `utils.mjs` has `getProjectDir()` with path traversal protection
- [ ] `run-hook.mjs` validates hookName against path traversal
- [ ] Changes applied to both root and template (template-sync rule)
