# Bug Fix: [BUG-103] Security Bypasses — Regex Gaps in pre-bash

**Status:** queued | **Priority:** P1 | **Date:** 2026-02-16
**Bug Hunt Report:** BUG-101-bughunt.md

## Symptom
Protection regexes in `pre-bash.mjs` can be bypassed with common command variations.
False positives block legitimate commands.

## Root Cause

### F-011: git clean bypass (P1)
Regex: `/git\s+clean\s+(?!-\w*n)-[a-z]*f[a-z]*d|...`

Only matches combined short flags (`-fd`, `-fxd`). Bypasses:
```bash
git clean --force -d        # long-form --force not matched
git clean -f -d             # separated flags not matched
git clean -d --force        # long-form in different position
```

The regex `(-[a-z]*f[a-z]*d)` requires `f` and `d` in the SAME flag group after a single `-`.

### F-013: git merge false positive (P2)
Regex: `/git\s+merge/i`

No word boundary after "merge". Matches:
```bash
git merge-base develop main    # query command, not a merge
git mergetool                  # tool invocation, not a merge
git log --merges               # log filter, not a merge
```

### F-014: push-to-main false positive (P2)
Regex: `/git\s+push[^|]*\bmain\b/i`

`\bmain\b` matches "main" anywhere in command due to `-` being a word boundary in regex:
```bash
git push origin fix-main-menu     # BLOCKED: branch name contains "main"
git push origin main-page-refactor # BLOCKED: same reason
```

## Fix Approach

### F-011: git clean
Match `-f` and `-d` independently (both required, any order, any format):
```javascript
// Match: git clean with both force+directory, excluding dry-run
// Check that command has git clean AND has force flag AND has -d flag AND no -n
function isDestructiveClean(cmd) {
  if (!/git\s+clean\b/i.test(cmd)) return false;
  if (/\b-\w*n/i.test(cmd)) return false; // dry-run present
  const hasForce = /(?:^|\s)(?:--force|-[a-z]*f)/i.test(cmd);
  const hasDir = /(?:^|\s)-[a-z]*d/i.test(cmd);
  return hasForce && hasDir;
}
```

### F-013: git merge
Add word boundary and exclude compound commands:
```javascript
/git\s+merge\b(?![-_])/i
```
This excludes `merge-base`, `mergetool` (starts with merge-/merge[a-z]).

### F-014: push-to-main
Match "main" only as the branch argument (after remote name):
```javascript
// Match: git push [remote] main — where main is the refspec, not part of a branch name
/git\s+push\s+\S+\s+main\b/i
```
Or use negative lookbehind/ahead for `-`:
```javascript
/git\s+push[^|]*(?<![a-zA-Z0-9_-])main(?![a-zA-Z0-9_-])/i
```
