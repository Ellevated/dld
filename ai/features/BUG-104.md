# Bug Fix: [BUG-104] Logic & Consistency — Paths, LOC, Patterns

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-16
**Bug Hunt Report:** BUG-101-bughunt.md

## Symptom
Several hooks have logic issues that cause incorrect behavior in edge cases:
worktree paths broken, LOC enforcement inverted, regex gaps in file matching.

## Root Cause

### F-006: validate-spec-complete dead in DLD (P2)
`ai/` is in `.gitignore` → spec files never appear in `git diff --cached`. Hook runs but never finds spec files to check. Works for users who don't gitignore `ai/` but is dead code for DLD itself.

**Decision:** Keep the hook (useful for template users), but document the `.gitignore` caveat in a comment.

### F-007: pre-edit.mjs normalizePath inconsistency (P2)
Line 59 uses raw `process.env.CLAUDE_PROJECT_DIR || process.cwd()` without the path traversal protection in `getProjectDir()` from utils.mjs.

### F-008: minimatch character class broken (P2)
`utils.mjs:159` — the regex escape `[.+^${}()|[\]\\]` also escapes `[` and `]`. A glob pattern like `[abc]*.py` becomes `\[abc\]\*\.py` — character classes never work. The JSDoc claims support but implementation breaks it.

### F-009: extractAllowedFiles regex too restrictive (P2)
`utils.mjs:182` — regex requires a file extension (`\.[a-zA-Z0-9]+`). Paths without extensions (directories like `src/shared/`) and glob patterns (`**/*.py`) don't match. Only exact file paths with extensions are recognized.

### F-015: LOC check timing inverted (P2)
`pre-edit.mjs:126-149` — checks LOC BEFORE edit is applied:
- 399-line file → passes → edit adds 200 lines → 599 lines (no block)
- 401-line file → BLOCKED → even if edit would remove 100 lines → can't fix oversized files

### F-022: checkSyncZone broken in worktrees (P2)
`pre-edit.mjs:73-74` — uses relative `template/${relPath}` which depends on CWD. In a git worktree, CWD is the worktree directory where `template/` doesn't exist. Sync reminders never fire in worktrees.

## Fix Approach

### F-006: Add comment, no code change
Add a comment in validate-spec-complete explaining the `.gitignore` caveat.

### F-007: Use getProjectDir()
Replace raw `process.env.CLAUDE_PROJECT_DIR || process.cwd()` with `getProjectDir()` import from utils.

### F-008: Fix minimatch character class handling
In the regex escape step, exclude `[` and `]` from escaping. Then handle them as-is (they'll form regex character classes naturally):
```javascript
.replace(/[.+^${}()|\\]/g, '\\$&')  // removed [ and ] from escape list
```

### F-009: Relax extractAllowedFiles regex
Allow paths without extensions and glob patterns:
```javascript
// Before: /[`*\-]*\s*([a-zA-Z0-9_./@-]+\.[a-zA-Z0-9]+(?::\d+(?:-\d+)?)?)[`*]*/
// After:  /[`*\-]*\s*([a-zA-Z0-9_./@*-]+(?:\.[a-zA-Z0-9]+)?(?::\d+(?:-\d+)?)?)[`*]*/
```
Key changes: `*` added to path characters, extension made optional.

### F-015: Allow edits that reduce LOC on oversized files
```javascript
if (loc >= maxLoc) {
  // Don't block if the edit might reduce file size
  // Pre-edit hook can't know the result, so just warn (askTool) instead of blocking
  askTool(`File exceeds LOC limit!\n\n${relPath}: ${loc} lines (limit: ${maxLoc})...`);
  return;
}
```
This is already the current behavior (askTool, not denyTool). Re-reading the code — it uses `askTool` which is a SOFT block (ask confirmation). This is actually correct! The user can approve. **Reduced to P3 — not a real bug, just a documentation note.**

### F-022: Resolve template path from project root
```javascript
import { getProjectDir } from './utils.mjs';
// ...
const projectRoot = getProjectDir();
const templatePath = join(projectRoot, 'template', relPath);
```
