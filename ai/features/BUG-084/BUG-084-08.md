# Bug: [BUG-084-08] Orphan spec if spark interrupted between writes

**Status:** queued | **Priority:** P1 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
In `template/.claude/skills/spark/completion.md`, spec file creation and backlog entry are two separate operations (lines 32-39 Pre-Completion Checklist and lines 49-64 Backlog Entry Verification).

If Spark is interrupted (timeout, crash, Ctrl+C) AFTER creating spec file but BEFORE adding backlog entry:
- Spec file exists in `ai/features/`
- No entry in `ai/backlog.md`
- Autopilot never sees the task (only reads backlog)
- Work is invisible and lost

The verification at lines 49-64 tries to catch this, but only if Spark reaches that step. Interruption before that step = orphan.

## Root Cause
Two-phase commit without transaction safety:
1. Write spec file
2. Write backlog entry

If interrupted between 1 and 2, state is inconsistent.

## Fix Approach
Add verification step AFTER both writes complete:

After line 39 (Pre-Completion Checklist), add:

```markdown
## Post-Write Verification (MANDATORY)

After BOTH spec file and backlog entry are written:

```bash
# Verify backlog entry exists
grep "{TASK_ID}" ai/backlog.md

# If NOT found (exit code 1) → ADD NOW
# Do NOT proceed to auto-commit without backlog entry!
```

This catches orphans before auto-commit, when recovery is still possible.
```

Update the checklist item:
```markdown
7. [ ] **Post-write verification** — grep found TASK_ID in backlog
```

## Allowed Files
1. `template/.claude/skills/spark/completion.md` — add verification after line 39
