# Bug: [BUG-084-10] Task-loop Step 6 commit has no failure handling

**Status:** queued | **Priority:** P1 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
In `template/.claude/skills/autopilot/task-loop.md`, Step 6 COMMIT (lines 129-148) shows:

```bash
git add {files_changed}
git commit -m "{type}({scope}): {description}"
```

Then immediately proceeds to:
```markdown
**After commit:**
1. Log to Autopilot Log in spec file
2. Increment task counter: `current_task += 1`
3. Continue to NEXT TASK (back to Step 1)
```

**Problem:** No error handling for commit failure. If commit fails (pre-commit hook failure, disk full, locked repository), the code:
1. Doesn't log the commit (it didn't happen)
2. Increments task counter (thinks it succeeded)
3. Moves to next task (work is lost)

## Root Cause
Assumed happy path. No failure branch in commit step.

## Fix Approach
Add error handling after git commit:

```markdown
## Step 6: COMMIT

**All reviews passed. Commit the changes.**

```bash
# Attempt commit
git add {files_changed}
git commit -m "{type}({scope}): {description}"

# Check exit code
if [ $? -ne 0 ]; then
  echo "ERROR: Commit failed!"

  # Retry once (transient failures)
  sleep 2
  git commit -m "{type}({scope}): {description}"

  # Still failed?
  if [ $? -ne 0 ]; then
    echo "CRITICAL: Commit failed twice. Escalating to human."
    # Set status: blocked
    # Add "ACTION REQUIRED: commit failure" to spec
    exit 1
  fi
fi
```

**After commit:**
(existing steps)
```

## Allowed Files
1. `template/.claude/skills/autopilot/task-loop.md` — lines 129-148 (Step 6 COMMIT section)

## Definition of Done
- [ ] Error handling added after git commit
- [ ] Retry logic (max 1 retry) for transient failures
- [ ] Escalation to human if both attempts fail
- [ ] Status set to blocked on persistent failure
- [ ] Work not lost if commit fails
