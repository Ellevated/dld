# Bug: [BUG-084-17] Protected paths .includes() allows false-positive matches

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
Line 108 of `template/.claude/hooks/pre-edit.mjs` uses `.includes()` to check protected paths:
```javascript
if (relPath.includes(protectedPath)) {
```
This matches unintended subdirectories. For example, `tests/contracts/` would also match `src/new_contracts/test_file.py` because the string "contracts/" appears in the path.

## Root Cause
`.includes()` does substring matching without path boundary awareness. It doesn't check if the match is at a directory boundary, leading to false positives.

## Fix Approach
Change `.includes()` to `.startsWith()` with proper path boundary. This ensures we're matching directory prefixes, not arbitrary substrings.

Replace line 108:
```javascript
// BEFORE:
if (relPath.includes(protectedPath)) {

// AFTER:
if (relPath.startsWith(protectedPath)) {
```

This works because `PROTECTED_PATHS` array (line 26) already has trailing slashes: `['tests/contracts/', 'tests/regression/']`

## Allowed Files
1. `template/.claude/hooks/pre-edit.mjs` — fix protected path matching at line 108
