# Bug: [BUG-084-16] Allowed Files line number suffix stripped — entire file allowed

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
Line 177 of `template/.claude/hooks/utils.mjs` in `extractAllowedFiles()` strips `:line-range` suffix from file paths:
```javascript
p = p.replace(/:\d+(-\d+)?$/, ''); // Remove line number suffix
```
This means `src/foo.py:10-20` becomes `src/foo.py` and the entire file is allowed, not just lines 10-20.

## Root Cause
Line number ranges were added to Allowed Files for documentation purposes but hook doesn't enforce them. The regex strips the suffix, making it informational only. This creates false security — users think they're restricting to specific lines but entire file is editable.

## Fix Approach
Two options:
1. Add comment documenting that line numbers are informational only, not enforced
2. Implement line-range checking (complex: requires parsing old_string/new_string line positions)

Recommend Option 1: Add clear comment at line 177 explaining line numbers are documentation hints, not enforced constraints.

## Allowed Files
1. `template/.claude/hooks/utils.mjs` — add documentation comment at line 177

## Definition of Done
- [ ] Comment added explaining line numbers are not enforced
- [ ] Users understand line ranges are hints only
