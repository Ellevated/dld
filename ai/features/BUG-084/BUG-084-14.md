# Bug: [BUG-084-14] ./test script referenced 20+ times but not in template

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
Multiple files reference `./test` or `./test fast` command:
- Line 9-10 of `template/CLAUDE.md`
- Line 21-22 of tester.md
- Line 91 of task-loop.md
But no `test` script exists in template root, causing confusion for new users.

## Root Cause
The test script is project-specific (depends on stack: pytest, npm test, cargo test, etc.) and intentionally not included in template. However, docs don't explain this - they reference `./test` as if it already exists.

## Fix Approach
Two options:
1. Add note in CLAUDE.md Quick Start that `./test` script should be created during bootstrap
2. Create a minimal template test script stub with "TODO: configure for your stack"

Recommend Option 1: Document in CLAUDE.md that user creates `./test` after bootstrap, and add example in comments.

## Allowed Files
1. `template/CLAUDE.md` — add note about creating test script
2. `template/.claude/agents/tester.md` — clarify test command is user-defined

## Definition of Done
- [ ] CLAUDE.md documents test script creation as part of setup
- [ ] tester.md clarifies ./test is project-specific
- [ ] Users understand they need to create this file
- [ ] No new failures introduced
