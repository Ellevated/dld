# Bug: [BUG-084-20] Bug Hunt cost not warned before starting

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
Line 131 of `template/.claude/skills/spark/bug-mode.md` shows cost estimate (~$40-70) but it's buried in the Bug Hunt Mode section. When user triggers Bug Hunt, spark immediately launches 6 persona agents without confirming the cost.

## Root Cause
No explicit confirmation step before Phase 1a launch. Cost information exists but user isn't prompted to approve spending before work begins.

## Fix Approach
Add explicit cost confirmation step in bug-mode.md before Phase 1a. Update the flow to:

1. Detect Bug Hunt mode trigger
2. Calculate estimated cost based on scope
3. Display cost estimate and ask user: "Bug Hunt will cost ~$X. Proceed? (yes/no)"
4. If yes → launch Phase 1a
5. If no → fall back to Quick Bug Mode or cancel

Add new section after line 142 (before Phase 1a):

```markdown
## Pre-Flight Confirmation

Before launching Bug Hunt:
1. Calculate scope (file count, complexity estimate)
2. Estimate cost: ~$6-10 per agent × 9 agents = $54-90
3. Ask user: "Bug Hunt estimated cost: $XX-XX. Proceed?"
4. If declined → offer Quick Bug Mode or cancel
```
