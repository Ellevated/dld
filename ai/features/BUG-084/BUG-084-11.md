# Bug: [BUG-084-11] task-loop.md references non-existent pre-review-check.py

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
Step 3 of task-loop.md (PRE-REVIEW CHECK) executes `python scripts/pre-review-check.py {files_changed}` but this script doesn't exist in template, causing autopilot to fail when it tries to run the pre-review check.

## Root Cause
Line 60 of `template/.claude/skills/autopilot/task-loop.md` references a script that was never added to the template. The step is hardcoded as mandatory but the script is missing.

## Fix Approach
Make the pre-review check conditional: document that it runs only if the script exists, otherwise skip. This allows projects to optionally add the script while not breaking autopilot when it's absent.

Update lines 55-75 of task-loop.md to:
1. Add conditional note: "If `scripts/pre-review-check.py` exists, run it. Otherwise skip to Step 4."
2. Update decision tree to handle "script not found" case

## Allowed Files
1. `template/.claude/skills/autopilot/task-loop.md` — add conditional logic
