# Bug: [BUG-084-18] Force push regex bypassed with non-standard flag order

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
Lines 54 and 64 of `template/.claude/hooks/pre-bash.mjs` check for force push patterns:
```javascript
/git\s+push\s+(-f|--force(?!-with-lease))[^|]*\b(develop|main)\b/i
/git\s+push[^|]*\b(develop|main)\b[^|]*(-f|--force(?!-with-lease))/i
```
These match common flag positions but can miss variations like:
- `git push origin -f --no-verify develop`
- `git push --tags -f develop`

## Root Cause
Regex patterns check for flags either immediately after `push` or at the end, but don't handle flags interspersed with other arguments. An attacker or confused agent could bypass by placing `-f` in an unexpected position.

## Fix Approach
Improve regex to match `-f` or `--force` at ANY position in the command when pushing to protected branches. Use a more flexible pattern that looks for the branch name anywhere and the force flag anywhere.

Replace lines 54 and 64 with single comprehensive regex:
```javascript
/git\s+push\b.*?(-f\b|--force\b(?!-with-lease)).*?\b(develop|main)\b|\bgit\s+push\b.*?\b(develop|main)\b.*?(-f\b|--force\b(?!-with-lease))/i
```

## Allowed Files
1. `template/.claude/hooks/pre-bash.mjs` — improve force push detection regex (lines 54, 64)
