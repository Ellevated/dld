# Bug: [BUG-084-26] Pre-flight skip without user warning

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
Autopilot SKILL.md line 178 says "Skip if either check fails" during pre-flight, but there's no instruction to warn the user WHY a task was skipped. Silent skips lead to confusion ("why isn't my task running?").

## Root Cause
Lines 173-178 in autopilot/SKILL.md:
```markdown
## Pre-flight Check

Before taking a task:

1. **Status:** Must be `queued` or `resumed`
2. **Plan:** Must have `## Implementation Plan`

Skip if either check fails.
```

There's no step 3: "Output warning message to user explaining why task was skipped."

## Fix Approach
Add explicit warning output instruction after line 178:

**Before:**
```markdown
Skip if either check fails.
```

**After:**
```markdown
Skip if either check fails.

**When skipping, output to user:**
```
⚠️ SKIP: {TASK_ID} — {reason}

Status: {current_status} (expected: queued or resumed)
Plan: {has_plan ? "present" : "missing"}

Fix the issue and re-run autopilot.
```

This ensures user knows:
1. Which task was skipped
2. Why it was skipped (status wrong or plan missing)
3. How to fix it
```
