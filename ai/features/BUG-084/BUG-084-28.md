# Bug: [BUG-084-28] Power tier not explained in Quick Start

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
CLAUDE.md lines 28-35 show:
```markdown
## DLD Tier

**Current:** ⭐ Standard

**What's included:**
- MCP: Context7 + Exa
- Skills: spark, scout, audit, review
- Hooks: Safety validation

**Upgrade:** Run `./scripts/setup-mcp.sh --power` for Power tier (council, autopilot, planner)
```

But Quick Start section (lines 37-51) mentions Power tier without explaining what it adds or why someone would want it. User sees MCP setup instructions but doesn't know Standard vs Power difference.

## Root Cause
Line 50 in CLAUDE.md:
```markdown
> MCP enables `/scout` research with Exa (web search, deep research) and Context7 (library docs).
```

This explains MCP in general but doesn't mention that Power tier adds Memory + Sequential Thinking MCPs and enables additional skills (council, autopilot, planner).

Users might skip `--power` option without understanding the benefits.

## Fix Approach
Add brief explanation to Quick Start section after line 50:

**Before:**
```markdown
> MCP enables `/scout` research with Exa (web search, deep research) and Context7 (library docs).
```

**After:**
```markdown
> **Tier selection:**
> - **Standard** (default): Context7 + Exa — enables research and docs lookup
> - **Power**: Adds Memory + Sequential Thinking — unlocks `/council` (5-expert review) and `/autopilot` (autonomous execution)
>
> For Power tier, run: `./scripts/setup-mcp.sh --tier 3`
```

This clarifies the difference and lets users make an informed choice.
