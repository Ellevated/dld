# Bug: [BUG-084-13] CLAUDE.md references non-existent .claude/contexts/ directory

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
Lines 63-64 of `template/CLAUDE.md` in the Contexts table reference `.claude/contexts/{domain}.md` and `.claude/contexts/shared.md`, but the `.claude/contexts/` directory doesn't exist in template.

## Root Cause
The Contexts table documents conditional loading from a directory that's created during bootstrap (by scaffold or manually), not included in the template. This confuses new users.

## Fix Approach
Add clarifying note that contexts directory is created after `/bootstrap` when domains are defined. Update table entries to indicate "(created after /bootstrap)" or make them conditional on directory existence.

Modify lines 59-66 (Contexts section) to add note explaining contexts/ is project-specific.

## Allowed Files
1. `template/CLAUDE.md` — add note about contexts directory lifecycle

## Definition of Done
- [ ] Note added explaining contexts/ is created during bootstrap
- [ ] Users understand this is not a missing file error
- [ ] No new failures introduced
