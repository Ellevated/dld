# Bug: [BUG-084-23] git add ai/ can stage unintended files

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
Spark completion.md line 166 uses `git add ai/` which stages everything in ai/ directory. This can include unintended files like temporary research notes, incomplete specs, or debugging artifacts.

## Root Cause
Line 166 in completion.md:
```bash
git add ai/
```

This is too broad — it stages:
- Intended: spec file, backlog.md, diary entries
- Unintended: ai/research/, ai/.cache/, ai/tmp/, incomplete drafts

The comment on line 172-174 says "Only commits spec, backlog, diary" but the command doesn't match this intent.

## Fix Approach
Change line 166 from broad `git add ai/` to explicit paths:

**Before:**
```bash
# 1. Stage spec-related changes only
git add ai/
```

**After:**
```bash
# 1. Stage spec-related changes only (explicit paths)
git add ai/features/${TASK_ID}*.md ai/backlog.md
```

If diary was written during spark, that's a separate commit (handled by diary-recorder).

## Allowed Files
1. `/Users/desperado/dev/dld/template/.claude/skills/spark/completion.md` — line 166, narrow git add scope

## Definition of Done
- [ ] `git add` uses explicit paths instead of `ai/`
- [ ] Only spec and backlog are staged in spark completion
- [ ] No risk of staging unintended ai/ directory files
