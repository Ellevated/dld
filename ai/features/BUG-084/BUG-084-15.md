# Bug: [BUG-084-15] No concurrency protection for parallel autopilot instances

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
`template/.claude/skills/autopilot/safety-rules.md` warns about parallel safety (lines 5-11) but doesn't enforce file locking or atomic status check-and-set. Two autopilot instances can read "queued" status simultaneously, both proceed, and collide on the same task.

## Root Cause
Safety rules are documented but not technically enforced. The protocol relies on manual check (READ backlog → VERIFY status) which isn't atomic. No file locking or lock files prevent race conditions.

## Fix Approach
Add explicit warning in safety-rules.md about parallel execution risks and recommend running single autopilot instance at a time. Document that atomic locking is not currently implemented.

Alternative: Add note suggesting users implement external orchestration (GitHub Actions workflow, task queue) if running multiple autopilot instances.

Update lines 5-11 to add:
"WARNING: No file locking implemented. Run ONLY ONE autopilot instance at a time to prevent conflicts."

## Allowed Files
1. `template/.claude/skills/autopilot/safety-rules.md` — add concurrency warning
