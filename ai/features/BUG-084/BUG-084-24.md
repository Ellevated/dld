# Bug: [BUG-084-24] setup-mcp.sh lacks cross-platform checks

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
setup-mcp.sh runs MCP setup commands without validating that Claude Code CLI is installed and running. If `claude` command is missing, the script fails with cryptic errors instead of helpful guidance.

## Root Cause
Line 48 in setup-mcp.sh checks for `claude CLI`:
```bash
check_server "claude CLI" "command -v claude" || all_ok=false
```

But this check only runs in `--check` mode (line 164-166). During interactive setup (line 184 calls interactive_menu), the claude CLI check is skipped.

Users can run through entire menu, select tier, and only discover at line 73-77 (add_server execution) that `claude` command doesn't exist.

## Fix Approach
Add prerequisite check at the start of interactive_menu() and setup_tier():

**After line 116 in interactive_menu():**
```bash
interactive_menu() {
    # Check prerequisites before showing menu
    if ! command -v claude >/dev/null 2>&1; then
        echo -e "${RED}Error: Claude Code CLI not found${NC}"
        echo ""
        echo "Install Claude Code first:"
        echo "  https://claude.ai/code"
        echo ""
        exit 1
    fi

    echo -e "${BLUE}"
    # ... rest of menu
```

This ensures `claude` command exists before offering MCP setup options.
