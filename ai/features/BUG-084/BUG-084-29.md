# Bug: [BUG-084-29] Coder Allowed Files error has no fix instructions

**Status:** queued | **Priority:** P2 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
Pre-edit hook (pre-edit.mjs lines 96-104) denies file edits outside Allowed Files with this message:
```
File not in Allowed Files!

{relPath}

Spec: {spec_path}

Allowed files:
  - {file1}
  - {file2}

Add file to spec's ## Allowed Files or change approach.
```

The last line says "Add file to spec" but doesn't tell HOW — where in the spec, what format, what to do next.

## Root Cause
Lines 96-104 in pre-edit.mjs:
```javascript
denyTool(
  `File not in Allowed Files!\n\n` +
    `${relPath}\n\n` +
    `Spec: ${specPath || '(not found)'}\n\n` +
    `Allowed files:\n${allowedList}\n\n` +
    `Add file to spec's ## Allowed Files or change approach.`,
);
```

The message is technically correct but not actionable. User doesn't know:
1. How to edit the spec (manual edit? which tool?)
2. What format to use in ## Allowed Files
3. Whether to restart coder after adding the file

## Fix Approach
Enhance the deny message with step-by-step fix instructions.

**Replace lines 96-104:**
```javascript
denyTool(
  `File not in Allowed Files!\n\n` +
    `File: ${relPath}\n` +
    `Spec: ${specPath || '(not found)'}\n\n` +
    `Allowed files:\n${allowedList}\n\n` +
    `To fix:\n` +
    `1. Edit ${specPath || 'the spec'}\n` +
    `2. Find ## Allowed Files section\n` +
    `3. Add: \`${relPath}\` — {description}\n` +
    `4. Save and retry\n\n` +
    `Or change approach to use only allowed files.`,
);
```

This gives:
- Clear location to edit (spec path)
- Section name (## Allowed Files)
- Exact format to use (backticks + description)
- Next action (save and retry)
