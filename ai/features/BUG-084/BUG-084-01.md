# Bug: [BUG-084-01] run-hook.mjs dynamic import has no try-catch

**Status:** queued | **Priority:** P0 | **Date:** 2026-02-14
**Parent:** [BUG-084](BUG-084.md)

## Symptom
If any hook .mjs file has a syntax error, the entire hook runner crashes with an uncaught exception. This causes Claude Code operations to fail silently or hang.

## Root Cause
The dynamic import at line 43 of `template/.claude/hooks/run-hook.mjs` has no error handling:

```javascript
await import(pathToFileURL(hookPath).href);
```

If the imported hook has a syntax error or throws during initialization, the exception propagates uncaught and crashes the runner.

This violates ADR-004: "Hooks must never crash — a crashing hook breaks Claude Code."

## Fix Approach
Wrap the dynamic import in a try-catch block. On error:
1. Log the error via `logHookError` (already available in the file)
2. Exit with code 0 (fail-safe: allow operation to proceed)

This matches the pattern used elsewhere in the file (lines 28-36 for git commands).

## Allowed Files
1. `template/.claude/hooks/run-hook.mjs` — add try-catch around line 43
